# NextRep 后续计划

本文记录 `1.3.x` 代码基线之后的维护优先级。它描述计划，不代表对应能力已经交付。

## 当前基线

React Native Reusables 风格的 UI 迁移已经基本完成：

- 页面已使用 NativeWind 语义颜色和统一的主题 token。
- `Text`、`Button`、`Card`、`Input`、`Badge`、`Separator`、`Switch`、`ToggleGroup` 等基础组件已落地。
- 首页、看板、AI 教练、设置、动作库和主要弹窗已迁移到统一组件体系。
- 深色、浅色和跟随系统主题已接入同一套主题状态。
- 月历、训练复盘、身体指标和动作历史已完成一轮交互与性能整理。
- Web SQLite 兼容层、历史数据库字段迁移和 JSON 导入校验已实现。

复杂图表、月历、训练录入和品牌光效仍保留业务专用实现，不强制套用通用组件。

## P0：发布质量

- 增加 `typecheck`、`lint`、`test` 脚本并接入 GitHub Actions。
- 为训练解析、看板聚合、AI JSON 归一化和备份导入增加单元测试。
- 建立 Android 冒烟测试清单，覆盖新增、编辑、删除、打卡、备份和恢复。
- 在 macOS 与真实 iOS 设备上完成首次完整验证。
- 将应用内显示版本改为读取 Expo 配置，避免硬编码版本漂移。
- 为 `1.3.0` 补齐发布标签、构建产物和发布说明。

## P0：敏感数据保护

- 将 API Key 从普通 SQLite 字段迁移到平台安全存储。
- 导出备份时默认移除 API Key，并提供明确的敏感信息确认流程。
- 为旧备份和旧数据库提供兼容迁移。
- 在 AI 配置页展示服务端数据发送范围和存储风险。

## P1：可靠性

- 为 SQLite 迁移增加显式版本号和可回滚的迁移记录。
- 为 AI 请求增加更清晰的错误分类、重试策略和响应结构诊断。
- 处理导入后页面状态刷新，确保恢复数据后无需重启应用。
- 统一日期处理策略，减少本地时区与 UTC 日期切换造成的边界问题。
- 清理未被当前界面使用的聊天接口和 Markdown 相关依赖。

## P1：可用性

- 补齐按钮、输入框、切换项和图表的无障碍标签。
- 验证大字体、窄屏设备和横向内容的布局表现。
- 为训练录入增加更明确的校验与错误反馈。
- 改善空数据、加载失败和离线 AI 场景的恢复路径。

## P2：产品能力

- 支持训练模板和周期计划。
- 支持更多身体指标与自定义指标。
- 增加数据范围筛选和可分享的训练总结。
- 在保持本地优先的前提下，评估可选的加密同步方案。

## 每次发布的最低验收

```bash
npm ci
npx tsc --noEmit
```

手动验证范围：

1. Android 冷启动和数据库初始化。
2. 力量、有氧训练的新增、编辑和删除。
3. 日期切换、打卡和月度热力图。
4. 看板、身体指标和动作详情。
5. 三种主题模式。
6. AI 配置测试、建议生成和计划应用。
7. JSON 导出、覆盖导入和清空数据。
8. Web 启动及主要页面渲染。
