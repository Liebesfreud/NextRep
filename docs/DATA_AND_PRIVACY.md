# 数据与隐私说明

本文描述当前代码的实际行为，不构成第三方 AI 服务的隐私承诺。

## 本地保存的数据

NextRep 将以下信息保存在设备的 `nextrep.db` SQLite 数据库中：

- 力量和有氧训练记录。
- 动作库与分类。
- 每日打卡和 AI 估算热量。
- 体重、体脂等身体指标。
- 昵称、身高、年龄、性别、训练目标和目标指标。
- AI 服务名称、Base URL、模型、API Key 和 token 统计。

主题偏好单独保存在 AsyncStorage。

应用目前没有账号系统，也不会自动把整个数据库同步到 NextRep 自有服务器。

## 发送给 AI 服务的数据

只有在用户测试连接、生成 AI 建议或执行 AI 能耗估算时，应用才会请求所选的第三方 AI 服务。

根据功能不同，请求可能包含：

- API Key 和模型 ID。
- 昵称、身高、年龄、性别和训练目标。
- 当天或近期训练动作、重量、组数、有氧数据和日期。
- 近期体重与体脂数据。
- 用户动作库名称与分类。

服务地址由用户配置。第三方服务如何记录、使用和保留这些信息，取决于该服务的条款和设置。

## API Key 风险

当前版本将 API Key 作为普通文本保存在 SQLite 中，未使用 Android Keystore、iOS Keychain 或其他平台安全存储。

完整 JSON 备份也包含 API Key，包括当前多配置字段和旧版兼容字段。备份文件未加密。请勿将备份上传到公开仓库、公共网盘、问题报告或聊天记录。

在安全存储迁移完成前，建议：

1. 使用权限范围尽可能小、可随时撤销的独立 API Key。
2. 为服务设置消费额度和告警。
3. 只在可信设备上保存配置。
4. 分享备份前手动删除 `apiKey`、`aiApiKey` 和相关配置字段。

## 导入、导出与清空

- 导出会生成包含全部应用数据的 JSON 文件，并打开系统分享界面。
- 导入会校验基本结构，然后在单个事务中覆盖当前所有数据。
- 清空会删除训练、动作库、打卡、身体指标和个人配置。
- 删除应用或清除应用存储通常也会删除本地数据库，具体行为取决于平台。

导入和清空不可撤销。执行前应保留一份受保护的有效备份。

## 网络边界

核心记录和看板功能不要求连接外部服务。AI 功能需要网络，并会连接用户配置的 Base URL。Expo 开发工具、EAS 构建和依赖安装可能访问各自的开发服务，但不属于应用运行时的数据同步功能。

## 安全问题

报告安全问题时，请提供最小复现信息，并对日志、请求、截图和备份进行脱敏。不要在公开 Issue 中提交真实 API Key、完整用户资料或完整数据库备份。
