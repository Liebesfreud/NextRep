# 贡献指南

感谢你改进 NextRep。提交改动前，请先阅读[开发指南](docs/DEVELOPMENT.md)和[架构说明](docs/ARCHITECTURE.md)。

## 开发流程

1. 从 `main` 创建短期分支。
2. 使用 `npm ci` 安装锁定版本的依赖。
3. 让改动保持单一目的，避免夹带无关重构。
4. 完成 TypeScript 检查和受影响平台的手动验证。
5. 提交说明使用清晰的动词和范围，例如 `fix: 修复历史日期训练打卡状态`。

## 提交前检查

```bash
npx tsc --noEmit
git diff --check
```

仓库暂未配置自动化测试和 lint。涉及数据、日期、AI 响应解析或导入导出的改动，需要在 Pull Request 中写明手动验证步骤和结果。

## 代码约定

- 使用 TypeScript 严格模式，不用 `any` 绕过可表达的类型。
- 页面负责组合和交互状态，数据库访问放在 `db/services/`。
- 优先复用 `components/ui/` 和现有业务组件。
- 静态颜色、间距和圆角使用语义 token；运行时计算的图表颜色可以使用内联样式。
- 使用 `@/` 路径别名，避免深层相对导入。
- 异步页面加载需要处理卸载和竞态，遵循现有 sequence ref 模式。
- 不在日志、错误信息、测试数据或提交内容中加入真实 API Key。
- 数据模型变化必须同时更新 `db/schema.ts`、`db/client.ts` 的初始化/迁移逻辑及相关文档。

## Pull Request 内容

请包含：

- 改动目的和用户可见行为。
- 受影响的页面、服务或数据表。
- 已执行的命令和手动验证范围。
- UI 改动的深色、浅色截图。
- 数据迁移、隐私或兼容性风险。

## 问题报告

请提供平台、系统版本、应用版本、复现步骤、预期结果和实际结果。崩溃或 AI 请求问题应附上已脱敏的错误信息，不要公开 API Key 或完整备份文件。
